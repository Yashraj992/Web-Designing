What is the drawback of declaring methods directly in JavaScript objects?

  • Object methods are actions that can be performed on objects.

  • A method is a function definition stored as a property value.

  • It can get to be a bit too verbose, and if you want to change the function's name, you have to change all of the functions added to the prototype, too.

  • They don't share an instance-specific scope, so you can't really have private variables.
